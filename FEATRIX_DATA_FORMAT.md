# 📊 Featrix Data Format for Sphere Viewer

## Overview

The **Featrix Sphere Viewer** is designed to work seamlessly with data exported directly from the **Featrix Platform**. When you run an embedding analysis in Featrix, the platform automatically generates data in the exact format expected by the Sphere Viewer.

> **Important**: This data format is the **standard export format** from Featrix. You don't need to transform or modify the data - simply export from Featrix and use it directly in the Sphere Viewer.

## 🎯 Getting Data from Featrix

### Step 1: Run Your Analysis in Featrix
1. Upload your dataset to the Featrix platform
2. Configure your embedding analysis
3. Run the analysis to generate 3D coordinates and clustering

### Step 2: Export the Results
1. Once analysis is complete, export the results as JSON
2. Featrix will provide a JSON file in the exact format below
3. This JSON contains everything the Sphere Viewer needs

### Step 3: Use in Sphere Viewer
```html
<script>
window.myFeatrixData = /* paste your exported JSON here */;
</script>
<script src="sphere-viewer.js" data-use-window-data="myFeatrixData"></script>
```

## 📋 Complete Data Format Specification

### Root Object Structure
```json
{
  "session": { /* Session metadata */ },
  "coords": [ /* Array of 3D coordinate objects */ ],
  "entire_cluster_results": { /* Clustering analysis results */ }
}
```

## 🔍 Detailed Field Descriptions

### 1. Session Object
Contains metadata about the Featrix analysis session:

```json
{
  "session": {
    "session_id": "unique-session-identifier",
    "status": "done",
    "done": true,
    "failed": false,
    "created_at": "2024-07-18T12:00:00Z",
    "metadata": {
      "dataset_name": "Customer Analysis",
      "num_rows": 1000,
      "num_columns": 15,
      "analysis_type": "embedding",
      "algorithm": "umap"
    }
  }
}
```

**Fields:**
- `session_id` (string) - Unique identifier for this analysis
- `status` (string) - Analysis status: "done", "running", "failed"
- `done` (boolean) - Whether analysis completed successfully
- `failed` (boolean) - Whether analysis failed
- `created_at` (string) - ISO timestamp of analysis creation
- `metadata` (object) - Additional information about the dataset and analysis

### 2. Coordinates Array
Array of objects, one for each data point, containing 3D coordinates and original data:

```json
{
  "coords": [
    {
      "0": -2.5,              // X coordinate in 3D space
      "1": 1.3,               // Y coordinate in 3D space
      "2": 0.8,               // Z coordinate in 3D space
      "cluster_pre": 0,       // Pre-assigned cluster ID
      "__featrix_row_id": 1,  // Original row identifier
      "__featrix_row_offset": 0, // Row offset in original dataset
      
      // Original data columns (preserved from your dataset)
      "scalar_columns": {
        "age": 25,
        "income": 50000,
        "score": 0.85,
        "rating": 4.2
      },
      "set_columns": {
        "category": "Premium",
        "region": "North America",
        "status": "Active"
      },
      "string_columns": {
        "name": "John Doe",
        "description": "Customer description",
        "email": "john@example.com"
      }
    }
    // ... more coordinate objects
  ]
}
```

**Coordinate Fields:**
- `"0"`, `"1"`, `"2"` (numbers) - 3D coordinates (X, Y, Z) generated by Featrix embedding
- `cluster_pre` (integer) - Initial cluster assignment
- `__featrix_row_id` (integer) - Unique identifier for this data point
- `__featrix_row_offset` (integer) - Position in original dataset

**Original Data Fields:**
- `scalar_columns` (object) - Numeric columns from your original dataset
- `set_columns` (object) - Categorical/enum columns from your original dataset  
- `string_columns` (object) - Text/string columns from your original dataset

### 3. Clustering Results
Contains results of clustering analysis at different cluster counts:

```json
{
  "entire_cluster_results": {
    "2": {
      "cluster_labels": [0, 1, 0, 1, 0],
      "silhouette_score": 0.75,
      "n_clusters": 2,
      "algorithm": "kmeans"
    },
    "3": {
      "cluster_labels": [0, 1, 2, 1, 0],
      "silhouette_score": 0.68,
      "n_clusters": 3,
      "algorithm": "kmeans"
    },
    "4": {
      "cluster_labels": [0, 1, 2, 3, 0],
      "silhouette_score": 0.62,
      "n_clusters": 4,
      "algorithm": "kmeans"
    }
    // ... more cluster counts
  }
}
```

**Clustering Fields:**
- Key (string) - Number of clusters as string ("2", "3", "4", etc.)
- `cluster_labels` (array) - Cluster assignment for each data point
- `silhouette_score` (number) - Quality metric for this clustering
- `n_clusters` (integer) - Number of clusters
- `algorithm` (string) - Clustering algorithm used

## 🔧 How Featrix Generates This Data

### Embedding Process
1. **Data Ingestion** - Featrix accepts your raw dataset (CSV, JSON, database, etc.)
2. **Feature Engineering** - Automatic feature extraction and preprocessing
3. **Dimensionality Reduction** - Advanced algorithms (UMAP, t-SNE, PCA) create 3D coordinates
4. **Clustering Analysis** - Multiple clustering algorithms test different cluster counts
5. **Export Generation** - Results packaged in the format above

### What Featrix Preserves
- ✅ **Original Data** - All your original columns are preserved in `scalar_columns`, `set_columns`, `string_columns`
- ✅ **Data Types** - Automatic classification of numeric, categorical, and text data
- ✅ **Relationships** - 3D coordinates reflect similarity and patterns in your data
- ✅ **Clustering** - Multiple clustering solutions for exploration
- ✅ **Metadata** - Complete information about the analysis process

## 🎯 Using Your Featrix Export

### Method 1: Direct Window Assignment
```html
<script>
// Paste your entire Featrix export here
window.myFeatrixData = {
  "session": { /* your session data */ },
  "coords": [ /* your coordinates */ ],
  "entire_cluster_results": { /* your clusters */ }
};
</script>
<script src="sphere-viewer.js" data-use-window-data="myFeatrixData"></script>
```

### Method 2: JSON File
```html
<!-- Save your Featrix export as a .json file -->
<script src="sphere-viewer.js" data-featrix-data="my-analysis.json"></script>
```

### Method 3: Manual API
```javascript
// Load your Featrix export data
const featrixData = await fetch('/api/my-analysis').then(r => r.json());

// Initialize sphere viewer
const viewer = new window.FeatrixSphereViewer();
viewer.init({
  data: featrixData,
  containerId: 'my-sphere-container'
});
```

## ✅ Data Validation

The Sphere Viewer expects:
- ✅ `session` object with valid metadata
- ✅ `coords` array with numeric coordinate fields `"0"`, `"1"`, `"2"`
- ✅ Each coordinate object has `__featrix_row_id`
- ✅ `entire_cluster_results` with at least one clustering solution
- ✅ Valid JSON format
